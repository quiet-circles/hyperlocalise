# 文档发布指南

<Callout type="warning">
  请在更新此页面时保持 `projectId="docs"` 不变。
</Callout>

<Tabs items={["cli", "api"]}>
  <Tab value="cli">
    Run `hyperlocalise status --verbose` before publishing the docs portal.
  </Tab>
  <Tab value="api">
    API 用户应保留令牌名称 `HYPERLOCALISE_API_KEY`。
  </Tab>
</Tabs>

<Card title="Replacement rules" href="/docs/rules">
  替换句子，而不是 `title` 属性或 `{locale.toUpperCase()}` 表达式。
</Card>

<Card
  title="Nested checks"
  href="/docs/nested(checks)"
  icon={<Badge text="beta" />}
>
  <Callout type="info">
    保持[参考链接][mdx-ref]不变，并保留字面文本，如 `MDPH_0_END`。
  </Callout>

  - 第一层嵌套项：`hyperlocalise sync pull`
  - 第二个嵌套项目，带有<Badge text="inline" />标记
</Card>

备用路由：{locale === "vi-VN" ? "/vi-VN" : "/"} 在运行时计算。

使用<Badge text="stable" />是在发布分支被冻结时构建的。

> 代码块可以包含<Badge text="quoted" />和`inlineCode()`。
>
> ![引用的图表](https://example.com/assets/quoted(flow).png)

  <Step title="Prepare">
    Review the [CLI guide](https://example.com/docs/cli(reference)) before publishing.
  </Step>
  <Step
    title="Publish"
    icon={<Badge text="go" />}
  >
    发送文档页面，并保持`docs/[locale]/index.mdx`不变。
  </Step>
</Steps>
