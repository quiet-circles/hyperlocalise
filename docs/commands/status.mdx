---
title: "status"
description: "Report translation status per key/namespace/locale in CSV, with optional interactive TTY dashboard."
---

## Usage

```bash
hyperlocalise status [--config <path>] [flags]
```

## Flags

- `--config`: path to config file
- `--locale`: target locale filter (repeatable)
- `--output`: output format (`csv`)
- `--group`: filter by group name
- `--bucket`: filter by bucket name
- `-i`, `--interactive`: render interactive dashboard in TTY (requires `--output csv`)

## Status values

- `translated`: value is non-empty and not marked as draft
- `needs_review`: provenance is `origin=llm` and `state=draft`
- `untranslated`: value is empty

Friendly label mapping used in some CI dashboards:

- `ready` = `translated`
- `needs review` = `needs_review`
- `missing` = `untranslated`

## CSV schema

`status` emits CSV with this header:

`key,namespace,locale,status,origin,state`

Column definitions:

- `key`: translation entry key in the file
- `namespace`: source mapping path from the configured bucket file (`from`)
- `locale`: target locale for the row
- `status`: computed status (`translated`, `needs_review`, `untranslated`)
- `origin`: provenance source (`llm`, `human`, `unknown`, depending on metadata availability)
- `state`: provenance state (`draft`, `final`, or empty when unknown)

Row behavior:

- one row per `key` + `namespace` + `locale`
- sorted deterministically by `namespace`, then `key`, then `locale`

## CI checks from CSV

Write a status artifact:

```bash
hyperlocalise status --config i18n.jsonc --output csv > artifacts/status.csv
```

Fail if any entries are `untranslated`:

```bash
awk -F, 'NR > 1 && $4 == "untranslated" {count++} END {exit count > 0}' artifacts/status.csv
```

Fail if `needs_review` exceeds a threshold:

```bash
awk -F, 'NR > 1 && $4 == "needs_review" {count++} END {exit count > 25}' artifacts/status.csv
```

## Examples

```bash
hyperlocalise status --config i18n.jsonc --locale es-ES --output csv
```

```bash
hyperlocalise status --config i18n.jsonc --group default --output csv -i
```
