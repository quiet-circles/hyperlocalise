## Mục tiêu

Tạo một `i18n.jsonc`, chạy lập kế hoạch dịch một cách an toàn, tạo đầu ra và xem xét trạng thái.

## 1. Khởi tạo cấu hình

```bash
hyperlocalise init
```

Thao tác này ghi `i18n.jsonc` trong thư mục hiện tại của bạn.

## 2. Cấu hình thông tin xác thực nhà cung cấp

`hyperlocalise` tự động đọc thông tin xác thực từ `.env` và `.env.local` trong thư mục dự án hiện tại của bạn. Các biến shell đã được export vẫn có mức ưu tiên cao hơn.

Ví dụ:


```bash .env.local
OPENAI_API_KEY="your-openai-api-key"
```

## 3. Làm khô-chạy trước

```bash
hyperlocalise run --config i18n.jsonc --dry-run
```

Kiểm tra `planned_total`, `executable_total`, và được liệt kê `tasks`.

## 4. Thực thi việc tạo ra

```bash
hyperlocalise run --config i18n.jsonc
```

Phần này ghi bản dịch và cập nhật `.hyperlocalise.lock.json`. Để biết chi tiết, xem [Hợp đồng tệp khóa](/reference/lockfile-contract).

## 5. Trạng thái đánh giá

```bash
hyperlocalise status --config i18n.jsonc --output csv
```

Sử dụng `needs_review` các hàng để xác định các mục vẫn cần được con người tuyển chọn.

## 6. Đánh giá chất lượng bản dịch khi cần thiết

```bash
hyperlocalise eval run --eval-set ./eval-set.json --output ./eval-report.json
```

Sử dụng nội dung này khi so sánh các lựa chọn prompt/hồ sơ/nhà cung cấp hoặc trước khi triển khai các bản cập nhật bản dịch lớn.
