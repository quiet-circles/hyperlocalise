##Các trình giữ chỗ được hỗ trợ
- `{{source}}`: nguồn địa phương- `{{target}}`: ngôn ngữ đích- `{{localeDir}}`: để trống cho đầu ra ngôn ngữ gốc, nếu không là ngôn ngữ đích
##Ví dụ
###Tệp ngôn ngữ JSON
```jsonc
{
  "from": "packages/ui/lang/{{source}}.json",
  "to": "apps/web/lang/{{target}}.json"
}
```

###Mẫu ngôn ngữ với`[locale]`token
```jsonc
{
  "from": "lang/en.json",
  "to": "lang/[locale].json"
}
```


###Bản đồ đệ quy Mintlify với các ký tự đại diện
```jsonc
{
  "from": "docs/**/*.mdx",
  "to": "docs/{{localeDir}}/**/*.mdx"
}
```

Điều này giữ nguyên cấu trúc tệp nguồn và ghi các tệp dịch vào cùng đường dẫn tương đối dưới mỗi thư mục ngôn ngữ.
Với`source=en`và mục tiêu`fr`, `es`, và`zh`:

- `target=en`giải quyết`docs/{{localeDir}}/index.mdx`để`docs/index.mdx`
- `target=fr`giải quyết`docs/{{localeDir}}/index.mdx`để`docs/fr/index.mdx`
- `target=es`giải quyết`docs/{{localeDir}}/index.mdx`để`docs/es/index.mdx`
- `target=zh`giải quyết`docs/{{localeDir}}/index.mdx`để`docs/zh/index.mdx`

Sử dụng mẫu này khi các trang nguồn tiếng Anh của bạn nằm trực tiếp bên dưới`docs/`và các trang đã dịch nên được ghi vào thư mục ngôn ngữ.
##Mẫu tốt
-Giữ cây nguồn và cây đích riêng biệt.-Sử dụng tên bucket ổn định theo miền(`ui`, `marketing`, `emails`).
-Giữ một trách nhiệm cho mỗi ánh xạ tệp.
##Chống-mẫu thiết kế
-Viết`to`đường dẫn trở lại các thư mục nguồn.-Tái sử dụng cùng một đường dẫn đầu ra cho nhiều bucket.-Pha trộn các token địa phương với quảng cáo-thay thế chuỗi hoc bên ngoài mẫu.
##Kiểm tra nhanh
Trước khi chạy:
1. xác nhận`from`giải quyết cho ngôn ngữ nguồn,2. xác nhận`to`có thể được tạo cho từng địa phương mục tiêu,3. chạy`hyperlocalise run --dry-run`để kiểm tra các nhiệm vụ đã lên kế hoạch.
