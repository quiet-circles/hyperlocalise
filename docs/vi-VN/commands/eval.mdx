<Note>
Tính năng này đang ở giai đoạn thử nghiệm. Cờ, hình dạng đầu ra và hành vi có thể thay đổi giữa các bản phát hành. Chỉ sử dụng trong các quy trình làm việc được kiểm soát và ghim phiên bản trong CI.

Lược đồ báo cáo và hành vi chấm điểm có thể thay đổi khi quy trình đánh giá dần hoàn thiện.
</Note>

Sử dụng `eval` các lệnh để đo chất lượng bản dịch trên một bộ đánh giá đại diện và áp dụng ngưỡng chất lượng trong CI.

## Sử dụng

```bash
hyperlocalise eval run --eval-set <path> [flags]
hyperlocalise eval compare --candidate <path> --baseline <path> [flags]
```

## đánh giá chạy

Chạy một hoặc nhiều biến thể thử nghiệm trên tập đánh giá của bạn.

### Lệnh này thực hiện gì

1. Tải bộ dữ liệu eval từ `--eval-set`.
2. Mở rộng các biến thể thử nghiệm từ các hồ sơ, nhà cung cấp, mô hình và phần ghi đè lời nhắc bạn đã chọn.
3. Chạy từng trường hợp cho từng biến thể của thử nghiệm.
4. Ghi báo cáo JSON khi bạn truyền `--output`.
5. In một bản tóm tắt ngắn gọn cho mỗi-bảng tóm tắt thử nghiệm.

### Cờ

- `--eval-set`: đường dẫn đến tập dữ liệu đánh giá (`.json`, `.jsonc`, `.csv`) (bắt buộc)
- `--profile`: ghi đè tên hồ sơ (có thể lặp lại)
- `--provider`: ghi đè nhà cung cấp (có thể lặp lại)
- `--model`: ghi đè mô hình (lặp lại được)
- `--prompt-file`: ghi đè tệp prompt
- `--prompt`: ghi đè lời nhắc nội tuyến (loại trừ lẫn nhau với `--prompt-file`)
- `--output`: Đường dẫn báo cáo đầu ra JSON

### Các trường của bảng tóm tắt

- `score`: điểm chất lượng trung bình có trọng số cho thử nghiệm
- `pass_rate`: số lần chạy thành công / tổng số lần chạy cho thử nghiệm
- `placeholder_violations`: số lượng kiểm tra tính toàn vẹn placeholder cứng-không đạt do vi phạm
- `latency_ms`: độ trễ trung bình cho thử nghiệm

### Ví dụ

Chạy với các giá trị mặc định từ cấu hình của bạn và viết báo cáo:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.jsonc \
  --output ./artifacts/eval-report.json
```

Chạy ma trận các cấu hình và các ghi đè nhà cung cấp/mô hình:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.csv \
  --profile default \
  --profile fast \
  --provider openai \
  --provider anthropic \
  --model gpt-4.1-mini \
  --model claude-sonnet-4-5
```

Chạy với tệp nhắc bị ghi đè:

```bash
hyperlocalise eval run \
  --eval-set ./evalsets/core.json \
  --prompt-file ./prompts/translation-eval.txt
```

## đánh giá so sánh

So sánh một báo cáo ứng viên với một báo cáo cơ sở.

Sử dụng lệnh này trong CI để ngăn chặn suy giảm chất lượng.

### Cờ

- `--candidate`: đường dẫn JSON của báo cáo ứng viên (bắt buộc)
- `--baseline`: đường dẫn JSON báo cáo cơ sở (bắt buộc)
- `--min-score`: điểm trọng số tối thiểu của ứng viên
- `--max-regression`: mức suy giảm điểm số tối đa được phép từ baseline sang candidate

### Hành vi CI

Lệnh thoát với lỗi khi:

- điểm có trọng số của ứng viên thấp hơn `--min-score`, hoặc
- hồi quy điểm vượt quá `--max-regression`.

### Ví dụ

So sánh báo cáo và chỉ in các giá trị tóm tắt:

```bash
hyperlocalise eval compare \
  --candidate ./artifacts/eval-candidate.json \
  --baseline ./artifacts/eval-baseline.json
```

Thất bại CI nếu điểm ứng viên giảm xuống dưới `0.82` hoặc giảm hồi quy nhiều hơn `0.02`:

```bash
hyperlocalise eval compare \
  --candidate ./artifacts/eval-candidate.json \
  --baseline ./artifacts/eval-baseline.json \
  --min-score 0.82 \
  --max-regression 0.02
```
