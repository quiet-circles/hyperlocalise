`hyperlocalise`从以下位置读取提供者凭据：
-导出当前 shell 会话中的变量。- `.env`和`.env.local`您当前项目目录中的文件。
优先级：
-已完成-导出的 shell 变量保持为-是。-对于尚未导出的变量，`.env.local`覆盖`.env`.

示例：
<CodeGroup>
```bash Export
export OPENAI_API_KEY="your-openai-api-key"
export AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
export AZURE_OPENAI_API_KEY="your-azure-openai-api-key"
```

```bash .env.local
OPENAI_API_KEY="your-openai-api-key"
AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
AZURE_OPENAI_API_KEY="your-azure-openai-api-key"
```
</CodeGroup>

##安全指导
-优先使用环境变量而非硬编码的令牌。-保留`.env`和`.env.local`超出版本控制。-定期更换令牌，并将其权限限定为最小。
##本地和持续集成的凭证管理标准
对所有远程存储适配器使用此操作模型，包括 Smartling、Lokalise 和内部自定义适配器。
###本地开发
1. 只提交非-密秘配置(`*Env`变量名)至`i18n.jsonc`.
2. 将真实令牌存储在`.env.local` (仅限开发者机器).
3. 添加`.env`和`.env.local`至`.gitignore`.
4. 使用独立的开发和生产 TMS 凭证。
###持续集成环境
1. 将机密存储在您的CI秘密管理器中(例如，GitHub Actions Secrets).
2. 仅将密钥注入需要它们的同步任务中。3. 将 CI 密钥映射到适配器配置所需的环境变量。4. 使用环境-针对生产同步的限定密钥和分支保护。
###轮换和最小权限
-在固定时间表上轮换令牌(例如每90天)以及团队成员离职手续。-为每个TMS工作区或项目创建专用的CI服务用户。-将每个令牌限制为拉取/推送所需的最小API范围。-通过进行一次干运行来验证旋转-在删除之前的凭据之前运行同步。
###审计-友好的日志记录策略
-切勿将原始密钥打印到日志、调试输出、截图或问题评论中。-仅记录凭证源名称(`SMARTLING_USER_SECRET`, `LOKALISE_API_TOKEN`)，而非数值。-保持CLI输出为机器格式-可读格式(`--output json`或`--output csv`)用于审计跟踪。-根据您的保留政策存储 CI 日志并同步工件。
##TMS 适配器安全检查表
在启用写入之前使用此检查表-已启用的 CI 作业。
- [ ]所有适配器密钥均通过环境变量提供，而非内联配置。- [ ]CI 使用专用的服务凭据(无个人访问令牌).
- [ ]凭证范围仅限于所需的项目和操作。- [ ]轮班计划由团队记录和负责。- [ ]流水线日志不会暴露令牌值。- [ ]计划干燥-运行`sync pull`和`sync push`作业运行成功。- [ ]如果需要，生产写入同步前需手动审批。
相同的检查清单适用于 Smartling、Lokalise 以及基于存储适配器接口构建的自定义适配器。
