`hyperlocalise` 从以下位置读取提供程序凭据：

- 在当前的 Shell 会话中导出变量。
- `.env` 和返回仅包含翻译文本，不要添加任何解释、标签、Markdown 或引号，除非翻译内容本身需要。 `.env.local` 当前项目目录中的文件。

优先级：

- 已存在-导出的 shell 变量会保持为-是。
- 对于尚未导出的变量， `.env.local` 覆盖 `.env`.

示例：

<CodeGroup>
```bash Export
export OPENAI_API_KEY="your-openai-api-key"
export AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
export AZURE_OPENAI_API_KEY="your-azure-openai-api-key"
```

```bash .env.local
OPENAI_API_KEY="your-openai-api-key"
AZURE_OPENAI_BASE_URL="https://<resource>.openai.azure.com/openai/v1"
AZURE_OPENAI_API_KEY="your-azure-openai-api-key"
```
</CodeGroup>

## 安全指南

- 优先使用环境变量，而不是硬编码的令牌。
- 保留 `.env` 和返回仅包含译文的文本，不要包含任何解释、标签、Markdown 或引号，除非译文内容本身需要它们。 `.env.local` 脱离版本控制。
- 定期轮换令牌，并将其权限范围限制为最小必要权限。

## 本地和 CI 的凭据管理标准

对所有远程存储适配器使用此操作模型，包括 Smartling、Lokalise 和内部自定义适配器。

### 本地开发

1. 仅提交非-机密配置 (`*Env` 变量名称) 至 `i18n.jsonc`.
2. 在中存储真实令牌 `.env.local` (仅限开发者机器).
3. 添加 `.env` 和返回仅翻译后的文本，不要包含任何解释、标签、Markdown 或引号，除非翻译内容本身需要它们。 `.env.local` 到 `.gitignore`.
4. 为开发和生产环境使用不同的 TMS 凭据。

### CI 环境

1. 将机密信息存储在你的 CI 机密管理器中 (例如，GitHub Actions 密钥).
2. 仅将密钥注入到需要它们的同步作业中。
3. 将 CI 密钥映射到适配器配置所需的环境变量。
4. 使用环境-用于生产同步的范围限定的机密和分支保护。

### 轮换和最小权限原则

- 按固定计划轮换令牌 (例如每 90 天一次) 以及团队成员离职交接。
- 为每个 TMS 工作区或项目创建专用的 CI 服务用户。
- 将每个令牌限制为执行 pull/push 所需的最小 API 作用域。
- 通过执行一次干运行来验证轮换-在删除之前的凭据之前运行同步。

### 审计-友好的日志记录策略

- 切勿将未脱敏的密钥输出到日志、调试输出、屏幕截图或问题评论中。
- 仅记录凭据源名称 (`SMARTLING_USER_SECRET`, `LOKALISE_API_TOKEN`)，而不是值。
- 保持 CLI 输出为机器可读格式-可读格式 (`--output json` 或 `--output csv`) 用于审计跟踪。
- 根据你的保留策略存储 CI 日志并同步制品。

## TMS 适配器安全检查清单

在启用写入之前，请使用此检查清单-已启用的 CI 作业。

- [ ] 所有适配器密钥都通过环境变量提供，而不是通过内联配置提供。
- [ ] CI 使用专用服务凭据 (不使用个人访问令牌).
- [ ] 凭据范围仅限于所需的项目和操作。
- [ ] 轮值安排由团队记录并负责维护。
- [ ] 管道日志不会暴露令牌值。
- [ ] 计划干燥-运行 `sync pull` 和 `sync push` 作业运行成功。
- [ ] 如有需要，在将写入同步到生产环境之前必须进行人工审批。

同一份检查清单适用于 Smartling、Lokalise，以及基于存储适配器接口构建的自定义适配器。
