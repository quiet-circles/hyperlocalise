## 拉取冲突

典型情况：本地策划的值与远程值不同。

安全解决方案：

1. 运行 `sync pull` 带有 `--output json`
2. 检查冲突条目
3. 决定真实来源
4. 使用更新的本地/远程状态重新运行

`--apply-curated-over-draft` 仅当本地条目为草稿时适用。

## 推送冲突

典型情况：本地草稿与远程精选值不同。

安全分辨率：

1. 以 JSON 格式检查冲突报告
2. 保持默认的冲突失败行为
3. 使用 `--force-conflicts` 仅适用于已审核的异常

## 推荐默认值

- 保持 `--fail-on-conflict=true`
- 保持 `--dry-run=true` 直到报告被审核
- 在持续集成中归档JSON报告以备审计追踪
