## 目标

创建一个 `i18n.jsonc`安全地运行翻译规划，生成输出，并查看状态。

## 1. 初始化配置

```bash
hyperlocalise init
```

这会写入 `i18n.jsonc` 在当前目录中。

## 2. 配置提供商凭据

`hyperlocalise` 自动从中读取凭据 `.env` 和返回仅包含译文的文本，不要包含任何解释、标签、Markdown 或引号，除非译文内容本身需要。 `.env.local` 在你当前的项目目录中。已导出的 shell 变量仍然优先生效。

## 3. 干燥-先运行

```bash
hyperlocalise run --config i18n.jsonc --dry-run
```

检查 `planned_total`, `executable_total`以及列出的 `tasks`.

## 4. 执行生成

```bash
hyperlocalise run --config i18n.jsonc
```

这会写入翻译并更新 `.hyperlocalise.lock.json`。有关详细信息，请参阅 [锁文件契约](/reference/lockfile-contract).

## 5. 审核状态

```bash
hyperlocalise status --config i18n.jsonc --output csv
```

使用 `needs_review` 行用于识别仍需要人工整理的条目。

## 6. 必要时评估翻译质量

```bash
hyperlocalise eval run --eval-set ./eval-set.json --output ./eval-report.json
```

在比较提示词/配置文件/服务提供商选项时，或在推广大规模翻译更新之前，请使用此内容。
