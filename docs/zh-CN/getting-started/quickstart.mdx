##目标
创建一个`i18n.jsonc`安全地进行翻译规划，生成输出，并审查状态。
##1. 初始化配置
```bash
hyperlocalise init
```

这写道`i18n.jsonc`在您当前的目录中。
##2. 配置提供程序凭据

##3. 干燥-先运行
```bash
hyperlocalise run --config i18n.jsonc --dry-run
```

检查`planned_total`, `executable_total`，及所列`tasks`.

##4. 执行生成
```bash
hyperlocalise run --config i18n.jsonc
```

这将编写翻译并进行更新`.hyperlocalise.lock.json`。详情请参见[锁定文件合约](/reference/lockfile-contract).

##5. 审核状态
```bash
hyperlocalise status --config i18n.jsonc --output csv
```

使用`needs_review`行以识别仍需人工策划的条目。
##6. 在需要时评估翻译质量
```bash
hyperlocalise eval run --eval-set ./eval-set.json --output ./eval-report.json
```

在比较提示/配置文件/提供者选项时或在推广大型翻译更新之前使用此功能。
