<Note>
此功能为实验性功能。标志、输出结构和行为可能会在不同版本之间发生变化。请在受控工作流中使用，并在 CI 中固定版本。

随着 TMS 集成不断成熟，冲突处理和同步契约可能会不断演进。
</Note>

## 用法

```bash
hyperlocalise sync push [--config <path>] [flags]
```

## 安全的默认行为

`sync push` 默认为干燥-运行模式和冲突失败。

## 标志

- `--config`: 配置路径
- `--locale`: 目标区域设置(s)，可重复
- `--dry-run`：仅预览 (默认 `true`)
- `--output`: `text` 或 `json`
- `--fail-on-conflict`：检测到冲突时返回错误 (默认 `true`)
- `--force-conflicts`：在策略允许的情况下，允许在不匹配的情况下覆盖 (default `false`)

## 示例

```bash
hyperlocalise sync push --config i18n.jsonc --locale es-ES --dry-run=false
```

参见 [冲突处理](/troubleshooting/conflicts) 使用前 `--force-conflicts`.
