<Note>
此功能为实验性功能。标志、输出结构和行为可能会在不同版本之间发生变化。请在受控工作流中使用，并在 CI 中固定版本。

随着 TMS 集成的成熟，冲突处理和同步契约可能会不断演进。
</Note>

## 使用情况

```bash
hyperlocalise sync pull [--config <path>] [flags]
```

## 安全的默认行为

`sync pull` 默认为 dry-运行模式。先查看报告，然后使用重新运行 `--dry-run=false`.

## 标志

- `--config`: 配置路径
- `--locale`: 目标区域设置(s)，可重复
- `--dry-run`：仅预览 (default `true`)
- `--output`: `text` 或 `json`
- `--fail-on-conflict`：检测到冲突时返回错误 (default `true`)
- `--apply-curated-over-draft`：允许经过审核的远程值替换本地草稿值 (默认 `true`)

## 示例

```bash
hyperlocalise sync pull --config i18n.jsonc --locale es-ES --output json
```

参阅 [冲突处理](/troubleshooting/conflicts) 用于解决策略。
