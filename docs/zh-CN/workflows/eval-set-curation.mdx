使用评估集来衡量对您最重要字符串的翻译质量。
##选择文件格式
您可以将评估集存储为 JSON、JSONC 或 CSV。
-当您需要嵌套结构和明确的元数据时，请使用 JSON 或 JSONC-当您想快速编辑类似于…列的电子表格时，请使用CSV`id`, `source`, `targetLocale`, `context`, `reference`, `tags`, `bucket`，和`group`
-在CSV中，使用分号分隔多个标签(`ui;short;icu`)

##选择具有代表性的覆盖范围
包含多种字符串类型，以便您可以检测不同内容形态中的回归。
-简短的用户界面字符串：按钮、标签、菜单项和简洁的错误文本-长-表单字符串：入职步骤、帮助文本、法律文案和交易消息-ICU 和复杂格式：复数规则、性别变体、选择语句，以及日期或数字格式的占位符-占位符和变量：诸如 令牌 之类`{name}`, `%s`，或`{{count}}`必须保持不变
##保持上下文紧密贴合每个案例
对于每个案例，存储一个稳定的ID并提供足够的上下文供审阅者参考。
-添加目标语言和源文本-包括截图、功能名称或意图说明`context`
-添加可选项`reference`当你已经有了可信的翻译时的文本-使用标签和可选的桶或组对病例进行标记，这样你就可以按区域拆分报告
##随着时间的推移保持质量
将评估集视为生产测试数据。
-当用户界面或产品文案发生变化时，审查并更新集合-移除不再映射到活动功能的过时案例-保持简单、中等和困难字符串的平衡-反复运行相同的集合以公平比较模型或提示的变化
