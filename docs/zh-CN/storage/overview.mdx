## 存储层的作用

存储连接：

- 本地翻译文件，
- sidecar 来源元数据，
- 远程适配器 API。

## 支持的适配器

- [Crowdin](/storage/crowdin)
- [LILT AI](/storage/lilt)
- [Lokalise](/storage/lokalise)
- [短语](/storage/phrase)
- [POEditor](/storage/poeditor)
- [Smartling](/storage/smartling)

## 条目标识

每个翻译条目由以下内容标识：

- `key`
- `context`
- `locale`

## 为什么存在边车元数据

你的用户-面向用户的文本保留在常规翻译文件中。操作元数据 (例如：草稿状态与精选状态) 保存在附属元数据文件中，使翻译内容保持干净。

## 拉取和推送生命周期

1. `run` 生成本地草稿 (`origin=llm`, `state=draft`).
2. `sync push` 将本地更改发送到远程存储。
3. 人工翻译人员在远程 TMS 中整理值。
4. `sync pull` 在本地应用经过筛选的更改。

## 冲突规则

- Pull 可以自动-应用精选内容-结束-启用时草稿。
- Push 默认保护精选的远程值。
- `--fail-on-conflict` 防止静默漂移。
- `--force-conflicts` 为 opt-用于存在风险的覆盖路径。

有关特定冲突决策，请参见 [冲突](/troubleshooting/conflicts).
